SimplifyIT - AI Jira Ticket Automation Tool

May 2024 – Present

During my tenure at General Motors, I participated in their company-wide hackathon and completed a project that integrated a basic Flask backend with a straightforward UI designed as a chat interface. The application utilized LLama3 chat and code models provided for the hackathon.

Users initiated conversations by specifying whether they intended to create a feature or a defect ticket. The chatbot then generated a corresponding template beside the chat window and proceeded to ask users questions to populate the template. Some of these questions included things like the application being worked on, instance of the application (cloud, on-prem, etc.) and after receiving responses to these questions, the codebase was queried and fed to the model to improve context and give the AI a look into how the code is written. Once both the chatbot and user completed filling out the template, it appeared filled out on the screen. The user was then prompted to click a "Finalize Ticket" button to create the ticket in the Jira dashboard. The bot would then create a file of test cases to ensure the feature/defect was implemented correctly.

The project won first overall, and I pitched the idea to Executive Directors and VPs of software at GM, getting the OK to continue development on a production version integrated into Jira's dashboard.

PlanIT - Trello Clone

Feb 2024 – Present

During my final exam for a class, I developed a Trello clone, a project management platform aimed at improving team collaboration and task management.

The Trello clone project replicates core functionalities of Trello, allowing users to create boards, lists, and cards to manage tasks and projects effectively. It was developed to demonstrate proficiency in full-stack web development and project management principles.

Key features include:

  • Board Management: Create, edit, and delete boards.
  • List Management: Add, reorder, and delete lists within boards.
  • Card Management: Create cards within lists, drag-and-drop cards between lists, and update card details.
  • User Authentication: Register new users, authenticate existing users, and manage user sessions.
  • Responsive UI: Designed with a responsive layout to ensure usability across various devices.

Technologies used include HTML, CSS, JavaScript for the frontend, and Python Flask and SQLite for the backend. The project was deployed on Google Cloud (no longer up).

Future improvements could include real-time updates using WebSocket technology, enhanced UI/UX design, integration with external APIs, addition of advanced features like file attachments and comments, and performance optimization for scalability.

This project was inspired by Trello and developed as a final exam project for a class. I plan to continue improving it and welcome contributions and feedback.
